## Key topics

- **Databricks AutoML** — automated model selection and feature engineering
- **MLflow** — experiment tracking, model registry, and serving
- **Feature Store** — centralised feature management and reuse
- **Cluster configuration** for ML workloads

## AutoML

Databricks AutoML trains a baseline model on your dataset with minimal
configuration. It produces a Python notebook for each candidate model so you
can inspect and modify the training code.

```python
from databricks import automl

summary = automl.classify(
    dataset=spark.table("churn_features"),
    target_col="churned",
    timeout_minutes=30,
)
print(summary.best_trial)
```

## MLflow tracking

Every AutoML run is automatically tracked in MLflow. You can also log your own
experiments:

```python
import mlflow

with mlflow.start_run():
    mlflow.log_param("max_depth", 5)
    mlflow.log_metric("accuracy", 0.92)
    mlflow.sklearn.log_model(model, "model")
```
